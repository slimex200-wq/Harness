# DailyLoop - Flutter 앱 설계 문서

> PRD: M11.md 기반 | Stack: Flutter + Dart + Firebase | Architecture: MVVM + Repository | State: Riverpod

---

## 1. Architecture Overview

3-layer MVVM 구조. 의존성 방향: `ui/ -> domain/ -> data/`, DI는 Riverpod Provider로 처리.

```
lib/
  core/           # 공통 유틸, 상수, 확장 함수
  data/           # Repository 구현, Firestore 데이터소스, DTO
  domain/         # Entity, Repository 인터페이스, UseCase (선택적)
  ui/             # Screen, Widget, ViewModel (StateNotifier)
  di/             # Riverpod Provider 정의 (전역)
  app.dart        # MaterialApp.router 루트 위젯
  main.dart       # Firebase 초기화, ProviderScope 진입점
```

### 계층 책임

| 계층 | 책임 | 의존 |
|------|------|------|
| `ui/` | 화면 렌더링, 사용자 입력 처리, ViewModel(StateNotifier) | `domain/` |
| `domain/` | Entity 정의, Repository 인터페이스 | 없음 (순수 Dart) |
| `data/` | Firestore CRUD, DTO <-> Entity 변환, 오프라인 캐시 | Firebase SDK |
| `di/` | Provider 선언, Repository 바인딩 | `data/`, `domain/` |
| `core/` | 날짜 유틸, 상수, 확장 함수, 에러 타입 | 없음 |

### 데이터 흐름

```
[Firestore] <-> [DataSource] <-> [Repository Impl] <-> [ViewModel/StateNotifier] <-> [UI Widget]
                                       ^
                                       |
                              [Repository Interface] (domain/)
```

- ViewModel은 `StateNotifier<AsyncValue<T>>`를 상속
- UI는 `ref.watch()`로 상태 구독, 로딩/에러/데이터 3상태 자동 처리
- Firestore 쓰기는 낙관적 업데이트: UI 즉시 반영 후 비동기 싱크

## 2. Data Model

### 2.1 Firestore Collections

```
users/{uid}/
  habits/{habitId}
    name: string                    # 습관 이름 (필수, 1-50자)
    icon: string                    # Material icon name (예: "fitness_center")
    color: string                   # hex color (예: "#FF5722")
    frequency: string               # "daily" | "weekdays" | "custom"
    customDays: number[]            # [0,1,2,3,4,5,6] (월=0, 일=6)
    reminderTime: timestamp | null  # 알림 시간
    createdAt: timestamp
    archivedAt: timestamp | null    # soft delete 시 설정
    order: number                   # 정렬 순서 (0부터)

  checks/{YYYY-MM-DD}
    completedHabits: string[]       # 완료된 habitId 배열
    updatedAt: timestamp
```

### 2.3 Firestore DTO Classes

#### `lib/data/dto/habit_dto.dart`

```dart
import 'package:cloud_firestore/cloud_firestore.dart';
import '../../domain/entity/habit.dart';

class HabitDto {
  final String id;
  final String name;
  final String icon;
  final String color;
  final String frequency;
  final List<int> customDays;
  final Timestamp? reminderTime;
  final Timestamp createdAt;
  final Timestamp? archivedAt;
  final int order;

  HabitDto({
    required this.id,
    required this.name,
    required this.icon,
    required this.color,
    required this.frequency,
    required this.customDays,
    this.reminderTime,
    required this.createdAt,
    this.archivedAt,
    required this.order,
  });

  factory HabitDto.fromFirestore(DocumentSnapshot doc) {
    final data = doc.data() as Map<String, dynamic>;
    return HabitDto(
      id: doc.id,
      name: data['name'] as String,
      icon: data['icon'] as String,
      color: data['color'] as String,
      frequency: data['frequency'] as String,
      customDays: List<int>.from(data['customDays'] ?? []),
      reminderTime: data['reminderTime'] as Timestamp?,
      createdAt: data['createdAt'] as Timestamp,
      archivedAt: data['archivedAt'] as Timestamp?,
      order: data['order'] as int,
    );
  }

  Map<String, dynamic> toFirestore() => {
    'name': name,
    'icon': icon,
    'color': color,
    'frequency': frequency,
    'customDays': customDays,
    'reminderTime': reminderTime,
    'createdAt': createdAt,
    'archivedAt': archivedAt,
    'order': order,
  };

  Habit toEntity() => Habit(
    id: id,
    name: name,
    icon: icon,
    color: color,
    frequency: HabitFrequency.values.byName(frequency),
    customDays: customDays,
    reminderTime: reminderTime?.toDate(),
    createdAt: createdAt.toDate(),
    archivedAt: archivedAt?.toDate(),
    order: order,
  );
}
```

## 5. Navigation Flow

### GoRouter 설정

```dart
// lib/router/app_router.dart
final appRouterProvider = Provider<GoRouter>((ref) {
  final authState = ref.watch(authStateProvider);

  return GoRouter(
    initialLocation: '/',
    redirect: (context, state) {
      final isLoggedIn = authState.valueOrNull != null;
      final isLoginRoute = state.matchedLocation == '/login';

      if (!isLoggedIn && !isLoginRoute) return '/login';
      if (isLoggedIn && isLoginRoute) return '/';
      return null;
    },
    routes: [
      GoRoute(
        path: '/login',
        name: RouteNames.login,
        builder: (context, state) => const LoginScreen(),
      ),
      ShellRoute(
        builder: (context, state, child) => MainShell(child: child),
        routes: [
          GoRoute(
            path: '/',
            name: RouteNames.home,
            builder: (context, state) => const HomeScreen(),
          ),
          GoRoute(
            path: '/stats',
            name: RouteNames.stats,
            builder: (context, state) => const StatsScreen(),
          ),
          GoRoute(
            path: '/settings',
            name: RouteNames.settings,
            builder: (context, state) => const SettingsScreen(),
          ),
        ],
      ),
      GoRoute(
        path: '/habit/new',
        name: RouteNames.habitNew,
        builder: (context, state) => const HabitFormScreen(),
      ),
      GoRoute(
        path: '/habit/:id/edit',
        name: RouteNames.habitEdit,
        builder: (context, state) => HabitFormScreen(
          habitId: state.pathParameters['id'],
        ),
      ),
    ],
  );
});
```

| **DataSource** | Firebase 예외 catch -> `AppException` 서브타입으로 변환. 원본 에러 코드 보존 |
| **Repository** | `AppException`을 그대로 전파. 추가 변환 없음 |
| **ViewModel** | `AsyncValue.guard()` 또는 try-catch로 `AppException` -> `state = AsyncError(e, st)` |
| **UI** | `ref.watch()` 결과의 `.when(error: ...)` 분기에서 `ErrorWidget` 또는 `SnackBar` 표시 |

## 10. Core Utilities

### 10.1 Streak Calculator

```dart
// lib/core/utils/streak_calculator.dart

/// 순수 함수. checks는 날짜 내림차순 정렬되어 있어야 함.
/// habit의 frequency에 따라 해당 요일만 카운트.
int calculateCurrentStreak({
  required Habit habit,
  required List<DailyCheck> checks,
  required DateTime today,
}) {
  int streak = 0;
  DateTime current = today;

  while (true) {
    if (!_isScheduledDay(habit, current)) {
      current = current.subtract(const Duration(days: 1));
      continue;
    }

    final dateStr = DateFormat('yyyy-MM-dd').format(current);
    final check = checks.firstWhereOrNull((c) => c.date == dateStr);

    if (check == null || !check.completedHabits.contains(habit.id)) {
      // 오늘이고 아직 안 했으면 어제부터 카운트
      if (current == today) {
        current = current.subtract(const Duration(days: 1));
        continue;
      }
      break;
    }

    streak++;
    current = current.subtract(const Duration(days: 1));
  }

  return streak;
}

bool _isScheduledDay(Habit habit, DateTime date) {
  return switch (habit.frequency) {
    HabitFrequency.daily => true,
    HabitFrequency.weekdays => date.weekday >= 1 && date.weekday <= 5,
    HabitFrequency.custom => habit.customDays.contains(date.weekday % 7),
  };
}
```

### 10.2 Date Extension

```dart
// lib/core/extensions/date_extension.dart
extension DateExtension on DateTime {
  String toDateString() => DateFormat('yyyy-MM-dd').format(this);

  String toKoreanDisplay() => DateFormat('M월 d일 EEEE', 'ko').format(this);

  DateTime get startOfWeek {
    final diff = weekday - DateTime.monday;
    return subtract(Duration(days: diff));
  }

  List<DateTime> get currentWeekDays {
    final start = startOfWeek;
    return List.generate(7, (i) => start.add(Duration(days: i)));
  }
}
```
