# V1-Web Strategy: Next.js + TypeScript Web App Harness

> Generation 1 (web) | Bootstrapped from v2 Flutter (65/70) | Target: 63/70+

## 학습 전이 (v2 Flutter → v1 Web)

v2 Flutter 하네스에서 65/70을 달성한 패턴을 웹 하네스에 직접 적용:

| v2 패턴 | 웹 적용 |
|---------|---------|
| SessionStart/Stop hooks | 동일 — session-state.json 기반 |
| 7-category eval rubric | 웹 전용 graders (eslint, vitest, next build) |
| Model routing (haiku/sonnet) | 동일 — read-only agents → haiku |
| Agent 참조 무결성 | CLAUDE.md 참조 = agents/*.md 1:1 매핑 |
| Eval command + skill 분리 | /web-eval command + web-eval skill |

## Model Routing Policy

| 태스크 | 모델 | 에이전트 |
|--------|------|---------|
| 코드 생성/수정/빌드 에러 | sonnet | build-error-resolver |
| 코드 리뷰 (read-only) | haiku | web-review-agent |
| 검증 (lint/test/build) | haiku | web-verify-agent |
| 보안 스캔 (read-only) | haiku | security-agent |
| 계획 분해 (read-only) | haiku | planner-agent |

## Eval Rubric (7 x 10 = 70)

| # | 카테고리 | Grader | 측정 |
|---|----------|--------|------|
| 1 | lint_compliance | code | `npx eslint . --max-warnings 0` |
| 2 | test_coverage | code | `npx vitest run --coverage` (80%+) |
| 3 | build_health | code | `npm run build` exit code |
| 4 | security_posture | code | `npx eslint-plugin-security` + secrets scan |
| 5 | arch_compliance | model | App Router 규칙, server/client 분리 |
| 6 | code_quality | model | TypeScript strict, naming, DRY |
| 7 | session_efficiency | model | 빌드 사이클, 불필요한 변경 |
